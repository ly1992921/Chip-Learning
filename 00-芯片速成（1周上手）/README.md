## Day 3：算力评估与选型逻辑

### 算力指标解读

| 指标 | 全称 | 含义 | 注意 |
|------|------|------|------|
| FLOPS | Floating Point Operations Per Second | 每秒浮点运算次数 | 分FP64/FP32/FP16/INT8 |
| TOPS | Tera Operations Per Second | 每秒万亿次操作 | 通常指INT8 |
| TFLOPS | Tera FLOPS | 每秒万亿次浮点运算 | 10^12 |
| PFLOPS | Peta FLOPS | 每秒千万亿次浮点运算 | 10^15 |

### 关键认知

**峰值算力 ≠ 实际性能**

```
实际性能 = 峰值算力 × 利用率

利用率通常：30% - 70%

为什么利用率不高？
1. Memory Wall（内存带宽瓶颈）
2. 数据依赖（等待数据）
3. 控制开销（调度、同步）
```

### Roofline模型（快速判断瓶颈）

```
如果 计算强度 > 内存带宽/算力 → Compute Bound（算力瓶颈）
如果 计算强度 < 内存带宽/算力 → Memory Bound（内存瓶颈）

大多数AI推理是 Memory Bound
→ 提升内存带宽比提升算力更有效
```

## Day 4：车规级芯片特别要求

### 功能安全等级（ISO 26262）

| ASIL等级 | 风险程度 | 典型应用 | 要求 |
|----------|----------|----------|------|
| ASIL-D | 最高 | 自动驾驶主控 | 最严格 |
| ASIL-B | 高 | ADAS | 较严格 |
| QM | 质量管理 | 信息娱乐 | 基础 |

### 车规认证

```
AEC-Q100     → 芯片可靠性（温度、寿命）
ISO 26262    → 功能安全（故障处理）
ASPICE       → 软件开发流程
ISO 21434    → 网络安全
```

## Day 5：芯片供应链与产业格局

### 产业链分工

```
芯片设计（Fabless）     → NVIDIA/高通/华为/地平线
    │
    ▼
晶圆制造（Foundry）     → 台积电/三星/中芯国际
    │
    ▼
封装测试（OSAT）        → 日月光/安靠/长电
    │
    ▼
终端应用               → 车企/手机厂商/数据中心
```

### 关键玩家

| 环节 | 全球龙头 | 中国玩家 | 特点 |
|------|----------|----------|------|
| 设计 | NVIDIA/高通/AMD | 华为/地平线/寒武纪 | 技术壁垒高 |
| 制造 | 台积电/三星 | 中芯国际 | 资本密集 |
| 设备 | ASML/Applied Materials | 北方华创 | 极度垄断 |
| 材料 | 信越/JSR | 沪硅产业 | 日本主导 |
| EDA | Synopsys/Cadence | 华大九天 | 美国垄断 |

### 地缘政治影响

```
美国出口管制 → 限制先进芯片/设备出口中国
    │
    ├── 影响：中国AI/高性能计算芯片获取
    │
    └── 应对：国产替代（华为昇腾/地平线/寒武纪）

关键风险点：
- 先进制程（7nm以下）受限
- EDA工具受限
- 先进封装（CoWoS）产能紧张
```

## Day 6-7：实战演练

### 模拟选型场景

**场景1：车载智驾芯片选型**
```
需求：L2+自动驾驶，需要处理8路摄像头
约束：车规级，功耗<50W，成本敏感

候选：
- NVIDIA Orin（性能强，贵，供应链风险）

□ 算力是否满足？

□ 供应链稳定性？
□ 成本是否符合预算？
□ 软件生态成熟度？
□ 技术支持能力？
```

- NVIDIA Jetson AGX Orin（性能强，贵）
